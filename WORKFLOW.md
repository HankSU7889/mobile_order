## 3. Item Name 解析规则

### 3.1 命名格式

```
{item_name} = "{系列}-{品牌}-{型号}-{颜色}"
```

**示例**：
| item_name | 系列 | 品牌 | 型号 | 颜色 |
|-----------|------|------|------|------|
| 水晶三星-S26-透 | 水晶 | 三星 | S26 | 透 |
| 潮酷苹果-iPhone15-黑 | 潮酷 | 苹果 | iPhone15 | 黑 |
| 商务华为-Mate60-透灰 | 商务 | 华为 | Mate60 | 透灰 |

### 3.2 解析代码

```python
def parse_item_name(item_name: str) -> dict:
    """
    解析 item_name，返回系列/品牌/型号/颜色
    
    规则：
    - 按 '-' 分割
    - parts[0] = 系列
    - parts[1] = 品牌
    - parts[2] = 型号
    - parts[3:] = 颜色（可能含多个'-'，用join保留）
    
    Args:
        item_name: 如 "水晶三星-S26-透"
    
    Returns:
        dict: {
            'series': '水晶',
            'brand': '三星', 
            'model': 'S26',
            'color': '透',
            'raw': '水晶三星-S26-透'
        }
    """
    if not item_name:
        return None
    
    parts = item_name.split('-')
    
    if len(parts) < 4:
        # 数据不规范，至少需要 4 段
        return None
    
    return {
        'series': parts[0],
        'brand': parts[1],
        'model': parts[2],
        'color': '-'.join(parts[3:]),  # 颜色可能含-
        'raw': item_name
    }
```

## 4. API 接口清单

### 4.1 产品查询类

| API | 方法 | 参数 | 返回值 | 说明 |
|-----|------|------|--------|------|
| `get_item_index` | GET | 无 | 完整索引 | 获取所有产品的系列/品牌/型号/颜色结构 |
| `get_series_list` | GET | 无 | 系列列表 | 获取所有系列（去重） |
| `get_brands_by_series` | GET | series | 品牌列表 | 获取指定系列下的品牌 |
| `get_models` | GET | series, brand | 型号列表 | 获取指定系列/品牌下的型号 |
| `get_colors` | GET | series, brand, model | 颜色列表 | 获取指定型号的所有颜色变体 |
| `search_models` | GET | keyword | 型号列表 | 搜索型号（模糊匹配） |
| `get_item_by_sku` | GET | item_code | 完整Item | 根据item_code获取Item详情 |

### 4.2 订单操作类

| API | 方法 | 参数 | 返回值 | 说明 |
|-----|------|------|--------|------|
| `submit_order` | POST | customer_name, sales_person, items | order_id | 提交订单 |
| `get_order_status` | GET | order_id | status | 查询订单状态 |
| `cancel_order` | POST | order_id, reason | success | 客户取消订单 |

### 4.3 后台管理类

| API | 方法 | 参数 | 返回值 | 说明 |
|-----|------|------|--------|------|
| `get_pending_orders` | GET | 无 | 订单列表 | 获取待审核订单 |
| `approve_order` | POST | order_id | success | 审核通过 |
| `reject_order` | POST | order_id, reason | success | 审核拒绝 |

### 4.4 业务员管理类

| API | 方法 | 参数 | 返回值 | 说明 |
|-----|------|------|--------|------|
| `get_sales_persons` | GET | 无 | 业务员列表 | 获取所有启用的业务员 |
| `create_sales_person` | POST | name, mobile | success | 创建业务员 |
| `update_sales_person` | POST | name, fields | success | 更新业务员 |
| `delete_sales_person` | POST | name | success | 删除业务员 |

## ❓ 常见问题

**Q: Item Name 格式不统一怎么办？**
A: 解析函数会跳过不符合格式的 Item，建议提前整理 ERP 数据

**Q: 9000+ SKU 加载慢怎么办？**
A: 使用索引缓存 + 分页加载，参考 `build_item_index()` 实现

**Q: 如何处理并发下单？**
A: ERPNext 本身支持事务，订单提交使用 `frappe.db.transaction` 保证数据一致性

**Q: 审核通过后如何生成 Sales Order？**
A: 使用 `erpnext.selling.doctype.sales_order.sales_order.make_sales_order` 或自定义 API
